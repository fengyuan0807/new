SKU数据获取

在获取sku数据时，我们在请求中包含了查询关键keyword，这时我么就需要对keyword进行判断，来返回不同的查询数据。

1. 接口设计

请求方式： GET /meiduo_admin/skus/?keyword=<名称|副标题>&page=<页码>&page_size=<页容量>

请求参数： 通过请求头传递jwt token数据。

返回数据： JSON

    {
        "counts": "商品SPU总数量",
        "lists": [
            {
                "id": "商品SKU ID",
                "name": "商品SKU名称",
                "spu_id": "商品SPU ID",
                "caption": "商品副标题",
                "category": "三级分类名称",
                "price": "价格",
                "cost_price": "进价",
                "market_price": "市场价格",
                "stock": "库存",
                "sales": "销量",
                "is_launched": "上下架",
                "specs": [
                    {
                        "spec_id": "规格id",
                        "option_id": "选项id"
                    },
                    ...
                ]
            },
            ...
        ],
        "page": "页码",
        "pages": "总页数",
        "pagesize": "页容量"
    }
返回值 类型 是否必须 说明
count int 是 SKUs商总量
lists 数组 是 SKU信息
page int 是 页码
pages int 是 总页数
pagesize int 是 页容量

2. API实现

1）在meiduo_admin/views目录下新建skus.py文件，并在该文件中实现API接口。

from django.db.models import Q

from rest_framework.permissions import IsAdminUser
from rest_framework.viewsets import ModelViewSet

from goods.models import SKU
from meiduo_admin.serializers.skus import SKUSerializer


class SKUViewSet(ModelViewSet):
    """SKU视图集"""
    permission_classes = [IsAdminUser]

    # 指定router动态生成路由时，提取参数的正则表达式
    lookup_value_regex = '\d+'

    def get_queryset(self):
        """获取当前视图所使用的查询集"""
        keyword = self.request.query_params.get('keyword')

        if keyword:
            skus = SKU.objects.filter(Q(name__contains=keyword) |
                                      Q(caption__contains=keyword))
        else:
            skus = SKU.objects.all()

        return skus

    # 指定序列化器类
    serializer_class = SKUSerializer

注意：当get /meiduo_admin/skus/时，视图集中的list方法会被调用，list方法中会直接调用self.get_queryset()获取视图所使用的查询集并序列化进行返回，而前端访问时可能传递keyword参数，所以此处需要将get_queryset进行重写。

2）在meiduo_admin/serializers目录下新建skus.py文件，并定义序列化器类。

from rest_framework import serializers

from goods.models import SKU
from goods.models import SKUSpecification


class SKUSpecSerializer(serializers.ModelSerializer):
    """商品规格信息序列化器类"""
    spec_id = serializers.IntegerField(label='规格id')
    option_id = serializers.IntegerField(label='选项id')

    class Meta:
        model = SKUSpecification
        fields = ('spec_id', 'option_id')


class SKUSerializer(serializers.ModelSerializer):
    """SKU商品序列化器类"""
    # 关联对象嵌套序列化
    category = serializers.StringRelatedField(label='三级分类名称')

    spu_id = serializers.IntegerField(label='SPU编号')

    # 关联对象嵌套序列化
    specs = SKUSpecSerializer(label='商品规格信息', many=True)

    class Meta:
        model = SKU
        # 排除模型的字段
        exclude = ('create_time', 'update_time', 'default_image', 'spu', 'comments')

注意： 当获取到一个sku对象之后，和sku关联的数据有3个：

  1. sku.spu: 和sku对象关联的spu对象。
  2. sku.category: 和sku对象管理的第三级分类对象。
  3. sku.specs: 和sku对象关联的sku规格信息查询集。

SKUSerializer中涉及到了category和specs关联对象的嵌套序列化操作。

3）在meiduo_admin/urls.py文件中进行路由配置。

# SKU商品管理
from rest_framework.routers import DefaultRouter
router = DefaultRouter()
router.register('skus', skus.SKUViewSet, base_name='skus')
urlpatterns += router.urls

总结：

  • SKU对象：
    • sku.spu: 和sku对象关联的spu对象
    • sku.category: 和sku对象关联的第三级分类对象
    • sku.specs: 和sku对象关联的sku规格信息查询集
