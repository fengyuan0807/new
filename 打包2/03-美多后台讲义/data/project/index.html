Django课程讲义

项目搭建

本小节目标：

  • 掌握美多前台项目代码的初始化操作
  • 掌握使用码云进行项目的源代码管理
  • 掌握后台管理项目子应用的创建和配置操作

1. 美多前台项目初始化

1）确保服务器中mysql数据库和redis都已经启动，如果还未启动则需要启动

ps aux | grep mysql：查看mysql数据库是否启动
ps aux | grep redis：查看redis数据库是否启动

2）使用pycharm打开提供的meiduo_mall美多前台项目代码，在终端执行命令创建项目所需的数据库

mysql -u<数据库用户> -p<数据库密码> -h <数据库服务器ip> < master_db.sql

注：创建的数据库用户表中添加了3个测试用户：
管理员账户：admin 密码：123456abc
普通用户账户：python和itcast 密码：123456abc

3）修改meiduo_mall项目目录下的dev.py配置文件，修改DATABASES配置项和CACHES配置项

注：

  1. 将DATABASES配置项中数据库的ip改为自己mysql数据库服务器的ip
  2. 将CACHES配置项中redis数据库的ip改写自己redis数据库服务器的ip

4）确保服务器中docker的tracker、storage容器已经启动，如果还未启动则需要启动

docker container ls：查看docker中正在运行的容器。
docker container start tracker storage：启动docker中的tracker和storage容器

5）运行美多前台项目，使用测试账户登录进行相关功能测试即可

python manage.py runserver

2. 码云仓库项目源代码管理

1）在码云上创建项目源代码仓库

2）负责码云仓库地址，克隆码云仓库到开发者本机，进行项目框架搭建

git clone <码云地址>

3）打开本地仓库目录，将meiduo_mall前台项目代码和meiduo_mall_admin后台管理前端vue代码放入

4）使用pycharm打开meiduo_project，编辑.gitignore文件，添加.idea忽略内容

# 忽略项目目录下的.idea目录及其内容
.idea

5）进入meiduo_project目录，在服务器本地进行提交

git add .
git commit -m '项目初始化'

6）将本地搭建的项目代码同步到远程码云仓库

git push origin master:master

3. 美多后台项目子应用创建和配置

美多后台是在原有美多商城项目的基础上创建一个meiduo_admin的子应用，在子应用中完成后台的所有功能。

1）进入meiduo_mall/meiduo_mall/apps目录，执行命令创建meiduo_admin子应用。

  python ../../manage.py startapp meiduo_admin

2）修改dev.py配置文件，注册meiduo_admin子应用。

 INSTALLED_APPS = [
     # ...
     'meiduo_admin.apps.MeiduoAdminConfig',
     # 注册rest_framework
     'rest_framework',
 ]

3）删除meiduo_admin下的views.py文件，并在meiduo_mall下创建views、serializers和utils目录。

views：保存定义API接口的py文件
serializers：保存定义序列化器类的py文件
utils：封装后端所需封装的工具类或工具函数

4）在meiduo_admin下创建urls.py，输入如下内容：

 from django.conf.urls import url

 urlpatterns = [
     # 进行url配置
 ]

5）在meiduo_mall/meiduo_mall/urls.py文件中包含meiduo_admin/urls.py文件，并添加url地址前缀。

 from django.conf.urls import url, include
 from django.contrib import admin

 urlpatterns = [
     # ...
     url('^meiduo_admin/', include('meiduo_admin.urls')),
 ]

6）在meiduo_admin/utils目录下创建exceptions.py，自定义DRF框架异常处理函数。

from rest_framework.response import Response
from rest_framework.views import exception_handler as drf_exception_handler
from rest_framework import status
from django.db import DatabaseError

def exception_handler(exc, context):
    # 先调用DRF框架的默认异常处理函数
    response = drf_exception_handler(exc, context)

    if response is None:
        # 补充数据库的异常处理
        if isinstance(exc, DatabaseError):
            response = Response({'detail': '数据库错误'}, status=status.HTTP_507_INSUFFICIENT_STORAGE)

    return response

7）在dev.py配置文件中设置DRF框架的异常处理函数。

REST_FRAMEWORK = {
    # 指定DRF框架的异常处理函数
    'EXCEPTION_HANDLER': 'meiduo_admin.utils.exceptions.exception_handler',
}
