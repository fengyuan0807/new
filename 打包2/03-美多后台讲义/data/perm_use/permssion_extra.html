权限控制拓展方案

本小节目标：

  • 了解权限控制的拓展方案

1. 权限控制问题说明

使用Django框架自带的权限控制时，开发时需要为每一个API接口添加permission_required装饰器进行权限控制的设置，权限控制时相对比较麻烦。

2. 权限控制拓展方案说明

以下提供一个权限控制的拓展方案，拓展方案相对于Django的权限控制设置，在权限设置上更加的简便、灵活。

1）权限数据表结构的更改。

注：以上6张数据表中，这里对权限数据表的结构进行了修改，权限数据表中不再有codename和content_type_id，而是换成了权限记录对应API接口的url地址和请求方式。

2）权限记录的添加

添加API接口对应的权限记录时，只需要指明对应API接口的url地址和请求方式即可。

举例：

id name url method
1 访问获取用户信息的API /meiduo_admin/users/ get
2 访问新增用户信息的API /meiduo_admin/users/ post
... ... ... ...

3）访问权限的控制

在用户访问API接口时，此处可以通过自定义DRF框架控制权限类在API接口调用之前对访问用户进行统一的权限检查，不需要再给每个API接口添加permission_required装饰函数。

权限控制类的伪代码如下：
