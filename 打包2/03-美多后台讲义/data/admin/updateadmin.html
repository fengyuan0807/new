更新管理员数据

本小节目标：

  • 理解指定管理员数据获取API接口的设计与实现
  • 理解指定管理员数据修改API接口的设计与实现

1. 获取指定的管理员数据

点击修改按钮时，我们需要先获取要修改的管理员数据。

1.1 接口设计

请求方式： GET /meiduo_admin/permission/admins/(?P<pk>\d+)/

请求参数： 通过请求头传递jwt token数据。

返回数据： JSON

    {
        "id": "用户id",
        "username": "用户名",
        "mobile": "手机号",
        "email": "邮箱",
        "groups": ['用户组id', ...],
        "user_permissions": ['权限id', ...]
    }
参数 类型 是否必须 说明
id int 是 用户id
username str 是 用户名
mobile str 是 手机号
email str 否 邮箱
groups list 是 用户组id
user_permissions list 是 权限id

1.2 API实现

注意：get /meiduo_admin/permission/admins/(?P<pk>\d+)/时，会调用AdminViewSet视图集中的retrieve方法，已经满足这里的需求，直接使用即可。

2. 修改指定的管理员数据

2.1 接口设计

请求方式： PUT /meiduo_admin/permission/admins/(?P<pk>\d+)/

请求参数： 通过请求头传递jwt token数据。

    {
        "username": "用户名",
        "password": "密码",
        "mobile": "手机号",
        "email": "邮箱",
        "groups": ['用户组id', ...],
        "user_permissions": ['权限id', ...]
    }
参数 类型 是否必须 说明
username str 是 用户名
mobile str 是 手机号
password int 否 密码
email str 否 邮箱
groups list 是 用户组id
user_permissions list 是 权限id

返回数据： JSON

    {
        "id": "用户id",
        "username": "用户名",
        "mobile": "手机号",
        "email": "邮箱",
        "groups": ['用户组id', ...],
        "user_permissions": ['权限id', ...]
    }
参数 类型 是否必须 说明
id int 是 用户id
username str 是 用户名
mobile str 是 手机号
email str 否 邮箱
groups list 是 用户组id
user_permissions list 是 权限id

2.2 API实现

1）修改meiduo_admin/serializers/permissions.py文件中的AdminSerializer序列化器类，重写update方法。

class AdminSerializer(serializers.ModelSerializer):
    """管理员用户序列化器类"""
    # ...

    def update(self, instance, validated_data):
        """修改管理员用户"""
        # 从`validated_data`中去除密码`password`
        password = validated_data.pop('password', None)

        # 修改管理员账户信息
        super().update(instance, validated_data)

        # 修改密码
        if password:
            instance.set_password(password)
            instance.save()

        return instance

注意：put /meiduo_admin/permission/admins/(?P<pk>\d+)/时会调用AdminViewSet视图集中的update方法，即是UpdateModelMixin中的update方法：
1）校验数据完成之后，此方法会调用序列化器对象.save()进行数据保存，save内部会调用AdminSerializer中的update方法进行数据保存，AdminSerializer虽然继承了ModelSerializer并且ModelSerializer中已经提供了update方法，但ModelSerializer中的update不满足这里的需要，所以需要在AdminSerializer中重写update方法来完成数据的保存。

总结：

  • 更新指定管理员数据时，需重写ModelSerializer中的update方法
