SKU图片新增

新增图片时，需要先获取sku商品的数据。

1. 获取SKU商品简单数据

1.1 接口设计

请求方式： GET /meiduo_admin/skus/simple/

请求参数： 通过请求头传递jwt token数据。

返回数据： JSON

    [
        {
            "id": "sku商品ID",
            "name": "sku商品名称",
        },
        ...
    ]
返回值 类型 是否必须 说明
id int 是 sku商品ID
name str 是 sku商品名称

1.2 API实现

1）在meiduo_admin/views/skus.py文件中实现API接口。

from meiduo_admin.serializers.skus import SKUSimpleSerializer

# GET /meiduo_admin/skus/simple/
class SKUSimpleView(ListAPIView):
    permission_classes = [IsAdminUser]

    # 指定视图所使用的查询集
    queryset = SKU.objects.all()

    # 指定视图所使用的序列化器类
    serializer_class = SKUSimpleSerializer

    # 注：关闭分页
    pagination_class = None

2）在meiduo_admin/serializers/skus.py文件中定义序列化器类。

class SKUSimpleSerializer(serializers.ModelSerializer):
    """SKU商品序列化器类"""
    class Meta:
        model = SKU
        fields = ('id', 'name')

3）在meiduo_admin/urls.py文件中添加路由配置。

urlpatterns = [
    # ...
    # 图片管理
    url(r'^skus/simple/$', skus.SKUSimpleView.as_view()),
]

3. Django框架文件保存的流程

在Django框架中，如果模型类中含有FileField或ImageField文件字段，

class TestModel(models.Model):
    file = models.FileField(...)
    # 或
    file = models.ImageField(...)

当通过模型新增或更新数据时，会自动调用默认文件存储类中的_save方法进行文件的保存，并将_save方法的返回值作为内容添加到对应表的文件字段中。

TestModel.objects.create(file='<file对象>')
或
obj = TestModel.objects.get(...)
obj.file = '<file对象>'
obj.save()

Django中的默认文件存储类是通过DEFAULT_FILE_STORAGE配置项指定的，其默认值为：

DEFAULT_FILE_STORAGE='django.core.files.storage.FileSystemStorage'

FileSystemStorage文件存储类中的_save方法默认将文件存储在MEDIA_ROOT指定的目录下方。

3.1 自定义文件存储类过程说明

如果你需要提供自定义文件存储：一个普遍的例子是在某个远程系统上储存文件，可以通过定义一个自定义的储存类来实现。

自定义文件存储类需要遵循以下步骤：

1）自定义储存类必须是django.core.files.storage.Storage的子类

from django.core.files.storage import Storage

class MyStorage(Storage):
    ...

2）Django必须能够不带任何参数来实例化你的储存类。 这意味着任何设置都应该从django.conf.settings中获取

from django.conf import settings
from django.core.files.storage import Storage

class MyStorage(Storage):
    def __init__(self, option=None):
        if not option:
            option = settings.CUSTOM_STORAGE_OPTIONS
        ...

3）你的储存类必须实现_save和exists方法，以及任何适合于你的储存类的其它方法。

注：

  1. 保存文件时，Django是调用文件存储类中的_save方法进行文件的保存。
  2. Django调用_save之前，会先调用文件存储类中的exists判断文件名和文件系统中原有的文件名是否冲突

3.2 自定义FDFS文件存储类

1）在meiduo_mall/utils/fdfs/storage.py文件中定义FDFSStorage文件存储类。

from django.core.files.storage import Storage
from django.conf import settings

from fdfs_client.client import Fdfs_client


class FDFSStorage(Storage):
    """FDFS自定义文件存储类"""
    def __init__(self, client_conf=None, base_url=None):
        if client_conf is None:
            client_conf = settings.FDFS_CLIENT_CONF

        self.client_conf = client_conf

        if base_url is None:
            base_url = settings.FDFS_URL

        self.base_url = base_url

    def _save(self, name, content):
        """
        name: 上传文件的名称
        content: 包含上传文件内容的File对象，content.read()获取上传文件内容
        """
        client = Fdfs_client(self.client_conf)

        # 上传文件到FDFS系统
        res = client.upload_by_buffer(content.read())

        if res.get('Status') != 'Upload successed.':
            raise Exception('上传文件到FDFS系统失败')

        # 获取返回的文件id
        file_id = res.get('Remote file_id')

        return file_id

    def exists(self, name):
        """
        判断上传文件的名称和文件系统中原有的文件名是否冲突
        name: 上传文件的名称
        """
        return False

    def url(self, name):
        return self.base_url + name

2）修改Django框架的默认文件存储类

DEFAULT_FILE_STORAGE='meiduo_mall.utils.fdfs.storage.FDFSStorage'

4. 新增图片数据

4.1 接口设计

请求方式： POST /meiduo_admin/skus/images/

请求参数： 通过请求头传递jwt token数据。

    {
        "sku_id": "sku商品ID",
        "image": "图片数据"
    }

返回数据： JSON

    {
        "id": "图片ID",
        "sku": "sku商品名称",
        "sku_id": "sku商品ID",
        "image": "sku图片地址"
    }
返回值 类型 是否必须 说明
id int 是 图片ID
name str 是 sku商品名称
sku_id int 是 sku商品ID
image str 是 sku图片地址

4.2 API实现

1）在meiduo_admin/serializers/skus.py文件中，修改SKUImageSerializer序列化器类，补充数据校验和create方法。

class SKUImageSerializer(serializers.ModelSerializer):
    """SKU图片序列化器类"""
    # 关联对象嵌套序列化
    sku = serializers.StringRelatedField(label='SKU商品名称')
    sku_id = serializers.IntegerField(label='SKU商品ID')

    class Meta:
        model = SKUImage
        exclude = ('create_time', 'update_time')

    def validate_sku_id(self, value):
        # sku商品是否存在
        try:
            sku = SKU.objects.get(id=value)
        except SKU.DoesNotExist:
            raise serializers.ValidationError('商品不存在')

        return value

    def create(self, validated_data):
        """sku商品上传图片保存"""
        # 调用ModelSerializer中的create方法
        sku_image = super().create(validated_data)

        # 保存上传图片记录
        sku = SKU.objects.get(id=validated_data['sku_id'])

        # 如果sku商品没有默认图片，则设置其默认图片
        if not sku.default_image:
            sku.default_image = sku_image.image
            sku.save()

        return sku_image

注意：post /meiduo_admin/skus/images时会调用SKUImageViewSet视图集中的create方法，即是CreateModelMixin中的create方法：

  1. 此方法会使用序列化器进行数据校验，SKUImageViewSet视图集使用的序列化器类为SKUImageSerializer，需要在SKUSerializer中补充validate_sku_id方法针对前端传递的sku_id进行数据校验。
  2. 校验数据完成之后，此方法会调用序列化器对象.save()进行数据保存，save内部会调用SKUImageSerializer中的create方法进行数据保存，SKUImageSerializer虽然继承了ModelSerializer并且ModelSerializer中已经提供了create方法，但ModelSerializer中的create不满足这里的需要，所有需要在SKUImageSerializer中重写create方法来完成数据的保存。
