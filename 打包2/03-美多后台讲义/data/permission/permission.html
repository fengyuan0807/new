权限控制说明

本小节目标：

  • 理解权限控制相关数据表的作用及其关系
  • 理解Django框架中用户权限分配的方式
  • 理解Django框架权限控制相关的模型类

在所有的管理系统中，一个必须有的功能就是对不同的用户进行不同的权限控制，在Django框架中，为了对用户的权限进行控制，提供了以下6张数据表。

1. 权限控制相关表

  • 权限表：保存权限记录数据。每条记录都代表一种操作权限，在使用Django框架时，在迁移生成每张数据表时，都会自动向控制权限表中添加3条记录，分别对应着数据表的增、删、改权限。
  • 用户表：保存用户信息数据。Django框架中，普通用户和管理员都存储在用户表中，通过is_staff进行区分，如果is_staff=True，说明用户是管理员，否则就是普通用户，如果用户is_staff=True并且is_superuser=True，说明用户是超级管理员，拥有所有的权限。
  • 用户组表：保存用户组的数据。

注：对于以上3张表，两两之间都是多对多的关系。

2. 用户权限分配

  • 在进行用户进行权限分配时，可以给用户组分配权限，然后给用户分配组，那么用户就拥有了所属组的权限，除此之外，也可以直接给用户分别特定的权限。

  • 用户的最终权限：组权限+额外权限。

3. 权限相关模型类

  • 用户模型类：users子应用下的User模型类。
    class User(AbstractUser):
  mobile = models.CharField(max_length=11)
  email_active = models.BooleanField(default=False)
  # ...
  • 权限模型类：auth子应用下的Permission模型类。
    # from django.contrib.auth.models import Permission
class Permission(models.Model):
  name = models.CharField(_('name'), max_length=255)
  content_type = models.ForeignKey(
      ContentType,
      models.CASCADE,
      verbose_name=_('content type'),
  )
  codename = models.CharField(_('codename'), max_length=100)
  • 权限内容类型模型类：contenttypes子应用下的ContentType模型类。
    # from django.contrib.contenttypes.models import ContentType
class ContentType(models.Model):
  app_label = models.CharField(max_length=100)
  model = models.CharField(_('python model class name'), max_length=100)
  • 用户组模型类：auth子应用下的Group模型类。
    # from django.contrib.auth.models import Group
class Group(models.Model):
  name = models.CharField(_('name'), max_length=80, unique=True)
  # ManyToManyField：多对多关系
  permissions = models.ManyToManyField(
      Permission,
      verbose_name=_('permissions'),
      blank=True,
  )

    注：如果两个模型类是多对多关系，可以在任意一端模型类中通过ManyToManyField定义多对多的关联属性，多对多关系中会自动生成中间的关系表。

总结：

  • 权限控制：控制不同的用户进行不同的操作。
  • 权限表、用户组表、用户表。
  • 最终权限：组权限+额外权限。
  • 多对多：ManyToManyField
