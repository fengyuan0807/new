美多管理后台

Django框架已经提供了一个Admin管理后台，但是Django本身Admin站点可修改的页面布局效果比较少，无法满足公司定制页面的需求，这时就需要独立开发一套后台管理系统，满足公司对后台数据管理的需求。

项目架构

开发模式：前后端分离

前端框架：VUE

后端框架：Django REST framework

功能部分：管理员登录，数据统计，用户管理，商品管理，订单管理，权限管理

主要技术：JWT用户认证，CORS跨域，FastDFS文件存储

美多后台前端页面目录结构

1）美多后台前端页面就是使用Vue-Cli创建的单页面系统项目，其主要目录如下：

  • src：前端开发的主要目录
    • components：存储除App.vue之外的其他单文件组件
    • router：存放路由规则文件
  • static：项目依赖的静态资源

2）如果美多后台前端需要调试运行，直接执行以下命令即可：

npm run dev

总结：

  • 美多管理后台：前后端分离开发模式，前端使用Vue.js，后端使用DRF框架
  • 美多管理后台前端页面：
    • 使用Vue-Cli创建的单文件组件项目
    • 运行：npm run dev
